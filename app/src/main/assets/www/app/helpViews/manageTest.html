This allows every organization location that has subscribed to allow users to view their default price lists. This also allows orgn users to print the default price lists. For this purpose, the fee schedules marked as residual for a fee type for a location for a service type are considered as the fee schedules to be considered for the price lists.

The various options that are discussed here include

a)       View Price Lists

b)      Print/PDF

Common Behavior and Options in the Forms for this module

 

  1. Welcome Mr. ………  (User->User Display Name) Size: 50 Alphanumeric. (This is a standard display in all forms and it may make sense to get this only once when the user logs in and use it till the user logs out)
  2. Logo – (Organization Location -> Logo Image file/ Organization-> Logo File) Size: 250 Alphanumeric. This shows the file location for the image file. This is linked to the organization location the user belongs to if the user belongs to the orgn_user table. If more than one orgn_user location exists for the user but the organization is the same, the logo shown will be the logo belonging to the organization. If the user belongs to different organizations or is not an orgn_user, the logo of the system organization to which the user is mapped will be taken. If the user is linked to more than one system organization, the system organization he/she was linked to first will be used for the logo.
  3. Picture of User – (User Additional Details->Photo Link Reference) Size: 250 Alphanumeric. This shows the file location for the image file.
  4. Id is not used but only the name functions as id for document types, transaction types, function names, parameter etc.
  5. In search screens, if no data is retrieved for the filter conditions set, the grid is displayed empty with pagination absent
  6. < moves the display to the previous page in the grid showing n records per page
  7. Show the number of pages retrieved in the pagination component
  8. Enter any number in the pagination component to move the display to the page number in the grid entered if it exists else this option is disabled I there is only 1 page.
  9. Any number from 1 to N is valid
  10. > moves the display to the next page in the grid showing n records per page if it exists else is disabled
  11. Back - goes to the previous screen from which the page was called
  12. Home – closes the form and Returns to Main Menu
  13. All date display will be based on the country settings for the country where the organization location is based as per its address.
  14. Form pagination for example 1 of 3. Here, we can enter the other page numbers to go to the respective page.  If we enter the same page number, no action is taken. If an invalid page number is entered, an error is reported.                                                                                                                           
  15. Form pagination               >             goes to the next page if there is a valid page else is disabled
  16. Form pagination               <             goes to the previous page if there is a valid page else is disabled

 

View Price Lists

 

Called From                        Main Menu-> Data Settings - >Price Lists

Parameter –Individual users can search the price lists for any organization but orgn users would be limited to the locations they have access to the privilege for.If no records are retrieved, “no records to view” would be displayed.
